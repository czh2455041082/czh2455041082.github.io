实训十 存储过程和触发器

一、实训目的与要求

通过本次实训，使学生加深对Mysq中存储过程的理解和使用。

二、实训设备和软件环境

计算机、MySQL软件、WorkBench或Navicat软件

三、实训课时

2课时

四、实训内容****和结果记录

1、创建存储过程select_stu，用指定的学号查询学生信息和成绩。

2、调用存储过程select_stu，查询学号为60231903的学生信息和成绩。

3、创建存储过程class_info，查询指定班级的男女生人数，输出班级信息，并根据男女生人数，输出相应信息’男生比女生多！’、’男女生人数一致！’或’男生比女生少!’。

4、调用存储过程 calss_info，查询W602319班的学生情况。

5、查看两个存储过程的定义。

6、删除定义的两个存储过程。

7、在student表中，定义一个触发器student_trigger，当一个学生的信息被删除时，把该生的信息添加到del_student表中，并删除对应的score表中相应的成绩，将删除的成绩添加到del_score表中。。

CREATE TABLE del_student ( studentno char(8) NOT NULL, name varchar(10) NOT NULL,

sex enum(‘男’,’女’) , birthday date DEFAULT NULL,

classNo char(7) DEFAULT NULL, PRIMARY KEY (studentno)

);

CREATE TABLE del_score ( studentno char(8) NOT NULL, courseno char(3) NOT NULL,

​ finalscore tinyint(4) DEFAULT ‘0’, PRIMARY KEY (courseno,studentno)

);

8、删除学生表中学号为60111901的学生信息，删除完成后查询del_student表和del_score表的内容。

9、查看student_trigger触发器的创建语句。

10、在score表上创建触发器score_update，当修改score表时，将用户信息、修改时间和操作类型记录到log_score表中。

​ op varchar(10) /操作类型，例如insert、delete和update/

11、在score表中将学号为60231903的同学的C03课程成绩修改为85，修改完成后，查看log_score表内容。

12、删除触发器student_trigger和score_update。
